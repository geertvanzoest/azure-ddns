Maak README.md in het Nederlands met de volgende secties in deze volgorde:

**1. Titel en overzicht (DOC-01)**
- Titel: `# ddns4j -- Azure DNS Dynamic Updater`
- Korte beschrijving: lightweight bash script dat als DDNS-client werkt voor Azure DNS
- Wat het doet: detecteert publiek IP via externe service, update A-record in Azure DNS via REST API
- Doelplatform: Raspberry Pi, zero dependencies buiten bash/curl/jq
- Badge voor CI status: `![CI](https://github.com/hscjeka/ns4j/actions/workflows/ci.yml/badge.svg)`

**2. Features (DOC-01)**
Bullet list met:
- Automatische IP-detectie met fallback (icanhazip.com -> checkip.amazonaws.com)
- Slimme update: alleen PUT als IP daadwerkelijk gewijzigd is
- Lock file (/tmp/ddns4j.lock) voorkomt gelijktijdige runs via flock
- Force modus (--force) voor update ongeacht IP-wijziging
- Debug modus (VERBOSE=1) voor uitgebreide logging
- Configureerbare TTL (DNS_TTL, default 300 seconden)
- Exit codes voor gestructureerde foutafhandeling (0-4)

**3. Quick Start (DOC-01)**
Genummerde stappen (kort, verwijst naar detail-secties):
1. Installeer dependencies (apt install jq)
2. Download script en maak uitvoerbaar
3. Maak Azure Service Principal aan (verwijs naar Configuratie)
4. Stel environment variables in
5. Test: `./ddns4j`
6. Stel cron job in

**4. Vereisten / Prerequisites (DOC-02)**
Tabel met:
| Dependency | Minimale versie | Controle commando |
| bash | >= 4.x | `bash --version` |
| curl | >= 7.68 | `curl --version` |
| jq | >= 1.6 | `jq --version` |

Plus: `sudo apt-get install jq` (enige niet-standaard dependency)

**5. Installatie (DOC-02)**
Stap-voor-stap met kopieerbare commando's:
```bash
# Download het script
sudo curl -o /usr/local/bin/ddns4j https://raw.githubusercontent.com/hscjeka/ns4j/main/ddns4j

# Maak uitvoerbaar
sudo chmod +x /usr/local/bin/ddns4j

# Controleer
ddns4j --help || echo "Geinstalleerd in $(which ddns4j)"
```

**6. Configuratie (DOC-03)**
Twee subsecties:

6a. Azure Service Principal aanmaken:
Stap-voor-stap met `az` CLI commando's (kopieerbaar):
```bash
# 1. App registratie aanmaken
az ad app create --display-name "ddns4j"

# 2. Service Principal aanmaken
az ad sp create --id <APP_ID>

# 3. Client secret genereren
az ad app credential reset --id <APP_ID> --display-name "ddns4j-secret"

# 4. DNS Zone Contributor rol toekennen (scope op zone niveau)
az role assignment create \
  --assignee <APP_ID> \
  --role "DNS Zone Contributor" \
  --scope "/subscriptions/<SUB_ID>/resourceGroups/<RG>/providers/Microsoft.Network/dnsZones/<ZONE>"
```

Uitleg: minimale rechten (alleen DNS zone, niet hele subscription).

6b. Environment variables instellen:
Tabel met alle 8 variabelen:

| Variabele | Verplicht | Beschrijving | Voorbeeld |
| AZURE_TENANT_ID | Ja | Microsoft Entra tenant GUID | `aaaabbbb-0000-cccc-1111-dddd2222eeee` |
| AZURE_CLIENT_ID | Ja | Service Principal application ID | `11112222-bbbb-3333-cccc-4444dddd5555` |
| AZURE_CLIENT_SECRET | Ja | Service Principal secret | `A1bC2dE3fH4iJ5kL6mN7oP8qR9sT0u` |
| AZURE_SUBSCRIPTION_ID | Ja | Azure subscription GUID | `00000000-0000-0000-0000-000000000000` |
| AZURE_RESOURCE_GROUP | Ja | Resource group van de DNS zone | `rg-dns` |
| DNS_ZONE_NAME | Ja | DNS zone naam (zonder trailing dot) | `example.com` |
| DNS_RECORD_NAME | Ja | Relatieve recordnaam | `home` |
| DNS_TTL | Nee | TTL in seconden (default: 300) | `300` |

Toon twee methodes voor instellen:
- Via `/etc/environment` (persistent, alle gebruikers)
- Via crontab environment (alleen voor cron)

Waarschuwing: sla secrets NOOIT op in het script zelf.

**7. Gebruik (DOC-04)**
Drie subsecties:

7a. Handmatig draaien:
```bash
# Normaal (update alleen als IP gewijzigd)
./ddns4j

# Force update (altijd updaten)
./ddns4j --force

# Debug modus (uitgebreide logging)
VERBOSE=1 ./ddns4j
```

7b. Cron job instellen:
```bash
# Open crontab
crontab -e

# Voeg toe (elke 5 minuten):
*/5 * * * * AZURE_TENANT_ID=xxx AZURE_CLIENT_ID=xxx ... /usr/local/bin/ddns4j >> /var/log/ddns4j.log 2>&1
```

Alternatief: env vars in /etc/environment en korter crontab commando.
Uitleg: log output naar bestand voor troubleshooting.

7c. Lock file:
Uitleg dat /tmp/ddns4j.lock via flock voorkomt dat twee instanties tegelijk draaien. Geen actie nodig van gebruiker.

**8. Troubleshooting (DOC-05)**
Tabel met alle exit codes:

| Code | Constante | Betekenis | Mogelijke oorzaken | Oplossing |
| 0 | EXIT_OK | Succes | - | Geen actie nodig |
| 1 | EXIT_CONFIG | Configuratiefout | Ontbrekende env var, jq niet geinstalleerd | Controleer alle verplichte env vars, installeer jq |
| 2 | EXIT_IP | IP-detectie mislukt | Geen internet, IP-services onbereikbaar | Controleer internetverbinding, test `curl https://icanhazip.com` |
| 3 | EXIT_AUTH | Authenticatie mislukt | Onjuiste credentials, verlopen secret, verkeerde tenant | Controleer AZURE_TENANT_ID/CLIENT_ID/CLIENT_SECRET, vernieuw secret |
| 4 | EXIT_DNS | DNS-operatie mislukt | Onvoldoende rechten, verkeerde zone/record naam | Controleer RBAC-rol, resource group, zone naam |

Plus subsectie "Debug modus":
```bash
VERBOSE=1 ./ddns4j
```
Uitleg: toont OAuth2 token status, IP-service responses, DNS record vergelijking.

Plus veelvoorkomende problemen:
- "Andere instantie draait, overgeslagen" -> lock file, wacht of verwijder /tmp/ddns4j.lock
- HTTP 401 bij token request -> client secret verlopen, vernieuw via `az ad app credential reset`
- HTTP 403 bij DNS update -> DNS Zone Contributor rol niet (correct) toegekend
